Initial mistakes & learnings

1) Take breaks: Divide your work into smaller chunks/milestones and take small breaks to avoid burnout.

Context: What I did was I took a challenge to complete my portfolio website under 48hrs and I did eventually manage to complete it in 56hrs but spent the next full day doing nothing. This should be avoided.

2) Making website Responsive: In CSS, we should try to use rem,%,vh so that we don't have to worry about how the website will look on the phone.

Context: I made my portfolio website and was very happy with how it looked on chrome on pc but when I opened the same website on my phone I was shocked and a little annoyed why this happened. Eventually, I understood how to fix this and came to know about the "toggle device toolbar" which we find in 'inspect' on chrome.

3)Learn more about shortcuts and emmet abbreviations.

Context: Let's take a particular case here - while creating a 'h1' tag if we also want a class inside it, I used to first make 'h1', then make a class in it and type everything out. But, later understood that this can be done by directly doing h1 followed by a dot then the class name, and then hitting enter. More shortcuts like this will make the process faster and the experience of writing code better.
